# Phenomes Analysis

FastAPI backend for pronunciation evaluation and listening comprehension practice.

## Overview

This repository implements a backend service that:

- evaluates spoken pronunciation using OpenAI Whisper and IPA phoneme scoring
- generates listening passages and comprehension questions
- evaluates spoken listening answers and returns relevance/correctness feedback
- stores user pronunciation profiles and phoneme performance in PostgreSQL

## Key Features

- Pronunciation evaluation with transcription, phoneme scoring, fluency, mistakes, and tips
- Listening module that generates passages, audio, and comprehension exercises
- User profile and recommendation endpoints
- Database persistence with SQLAlchemy and Alembic
- Audio conversion / transcription pipeline using Whisper and pydub

## Requirements

- Python 3.10+ (recommended)
- PostgreSQL
- FFmpeg installed on the host machine
- OpenAI API key

## Setup

1. Clone the repo

```bash
git clone <repo_url>
cd phenomes_analysis
```

2. Create and activate a virtual environment

```bash
python -m venv venv
venv\Scripts\activate
```

3. Install dependencies

```bash
pip install -r requirements.txt
```

4. Create a `.env` file in the repository root with at least:

```env
DATABASE_URL=postgresql://USER:PASSWORD@HOST:PORT/databasename
OPENAI_API_KEY=your_openai_api_key
```

Optional environment variables:

```env
LANGCHAIN_TRACING_V2=false
LANGCHAIN_API_KEY=
LANGCHAIN_PROJECT=phenome_analysis_agent
ENVIRONMENT=development
LOG_LEVEL=INFO
WHISPER_MODEL=medium.en
STATIC_AUDIO_DIR=app/static/audio
TEMP_DIR=temp
```

5. Run database migrations

```bash
alembic upgrade head
```

6. Start the FastAPI server

```bash
uvicorn app.main:app --reload
```

## API Endpoints

The application exposes the following endpoints. For full request/response documentation, see `endpoint_documentation.md`.

### Health

- `GET /healthz`
  - Response: `{ "status": "ok" }`

### Pronunciation

- `POST /test/analyze`
  - Upload form-data field: `file` (audio file)
  - Query parameters:
    - `user_id` (UUID)
    - `reference_text` (string, optional)
  - Returns pronunciation analysis, scores, phoneme details, mistakes, and tips

### Pronunciation Questions

- `GET /question/next`
  - Query parameter: `user_id` (UUID)
  - Returns a dynamically generated pronunciation question for the user

### Listening Module

- `GET /listening/module`
  - Query parameters:
    - `difficulty` (`easy`, `medium`, `hard`)
    - `num_questions` (1-10)
  - Returns a passage, audio URL, session ID, and comprehension questions

- `POST /listening/evaluate`
  - Form-data fields:
    - `session_id`
    - `audio_1` (file)
    - `audio_2` (file)
    - `audio_3` (file)
  - Returns evaluation results for the three submitted answers

### User Profile

- `GET /api/v1/pronunciation/profile/{user_id}`
  - Returns pronunciation profile data, current level, weak/strong phonemes, and progress

- `GET /api/v1/pronunciation/recommendations/{user_id}`
  - Returns practice recommendations based on weak phonemes and current level

## Development

- Use `pytest` to run tests
- The app loads a Whisper model at startup, which may require significant memory
- Static audio files are served from `/static`

## Notes

- The main FastAPI app entrypoint is `app/main.py`
- Configuration is centralized in `app/core/config.py`
- Database session dependency is in `app/db/session.py`
- Routes are defined under `app/routes/`

For additional endpoint details, refer to `endpoint_documentation.md`.
