🎧 Prounciation Analysis & Listening Evaluation System

A FastAPI-based backend system that evaluates both:

🗣️ Pronunciation (Speaking Skills)
🎧 Listening Comprehension

using OpenAI models, IPA phoneme analysis, and audio processing.

🚀 Key Features

🗣️ 1. Pronunciation Evaluation

Upload audio input
Transcribe using OpenAI
Compare reference text vs spoken transcript
Perform IPA-based phoneme analysis

Detect:
Mispronounced words
Missing/extra phonemes
Word-level differences

Generate:
Phoneme similarity score
Fluency score
Mistake list with severity
Improvement tips

🎧 2. Listening Comprehension Evaluation

Auto-generate listening passages
Auto-generate questions
Accept audio response
Transcribe & score using LLM

Metrics:

Relevance
Correctness
Feedback

🧠 3. AI-Based Question Generation
Dynamic passage creation
Structured question generation
Difficulty-based selection

🗄️ 4. Database Integration

Stores results for both modules:

Pronunciation
Listening

Using:

SQLAlchemy ORM
Alembic migrations

⚙️ 5. Audio Processing Pipeline

Upload
Convert (FFmpeg + pydub)
Transcribe
Score
Save to DB

## API Endpoints

The application exposes the following endpoints. For full request/response documentation, see `endpoint_documentation.md`.

### Health

- `GET /healthz`
  - Response: `{ "status": "ok" }`

### Pronunciation

- `POST /test/analyze`
  - Upload form-data field: `file` (audio file)
  - Query parameters:
    - `user_id` (UUID)
    - `reference_text` (string, optional)
  - Returns pronunciation analysis, scores, phoneme details, mistakes, and tips

### Pronunciation Questions

- `GET /question/next`
  - Query parameter: `user_id` (UUID)
  - Returns a dynamically generated pronunciation question for the user

### Listening Module

- `GET /listening/module`
  - Query parameters:
    - `difficulty` (`easy`, `medium`, `hard`)
    - `num_questions` (1-10)
  - Returns a passage, audio URL, session ID, and comprehension questions

- `POST /listening/evaluate`
  - Form-data fields:
    - `session_id`
    - `audio_1` (file)
    - `audio_2` (file)
    - `audio_3` (file)
  - Returns evaluation results for the three submitted answers

### User Profile

- `GET /api/v1/pronunciation/profile/{user_id}`
  - Returns pronunciation profile data, current level, weak/strong phonemes, and progress

- `GET /api/v1/pronunciation/recommendations/{user_id}`
  - Returns practice recommendations based on weak phonemes and current level

## Development

- Use `pytest` to run tests
- The app loads a Whisper model at startup, which may require significant memory
- Static audio files are served from `/static`

## Notes

- The main FastAPI app entrypoint is `app/main.py`
- Configuration is centralized in `app/core/config.py`
- Database session dependency is in `app/db/session.py`
- Routes are defined under `app/routes/`

For additional endpoint details, refer to `endpoint_documentation.md`.
